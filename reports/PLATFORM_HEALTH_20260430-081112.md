## Blocker / Major API issues (fix first)

### B1. `/api/user/career-interests` returns 401 for **every** authenticated role
- **Symptom:** Student, instructor, support, orgAdmin, superAdmin all get `{"error":"Not authenticated"}` even with a valid bearer token.
- **Likely cause:** `requireAuthInProd` middleware in `backend/server.js` line ~7300 isn't accepting bearer tokens (probably session-cookie-only). Compare to `authenticateBearerToken` used elsewhere.
- **Impact:** Career interests feature is broken for all logged-in users.
- **Suspect file:** [backend/server.js](../backend/server.js#L7300)

### B2. `/api/admin/students/search` returns **500** for orgAdmin and superAdmin
- **Symptom:** `{"error":"Failed to search students"}` — server-side exception.
- **Impact:** Admin student search is unusable (the endpoint backing the admin Students tab).
- **Suspect file:** [backend/server.js](../backend/server.js#L29844)

### B3. `/api/admin/students/export` returns **500** for orgAdmin and superAdmin
- **Symptom:** `{"error":"Failed to fetch student"}` (note the message references singular "student" but the route is the bulk CSV export — message bug + actual failure).
- **Impact:** Admin CSV export is broken.
- **Suspect file:** [backend/server.js](../backend/server.js#L30333)

### B4. `/api/admin/org-summary` returns **400** for superAdmin
- **Symptom:** `{"error":"No organization scope"}`.
- **Verdict:** May be intentional (super-admin has no org scope), but the admin UI almost certainly calls this on the super-admin dashboard. If so, route should accept an `?orgId=` query or be gated out of the super-admin UI.
- **Suspect file:** [backend/server.js](../backend/server.js#L21762)

### B5. Instructor role can read `/api/admin/reports/{readiness,activity,ged-results,domain-weaknesses}`
- **Symptom:** Instructor (non-admin) gets 200 on routes that the codebase labels as admin-prefixed.
- **Verdict:** Possibly intentional (instructors should see their class's reports) but inconsistent with other `/api/admin/*` routes that correctly reject non-admin callers. Confirm intent and align the gate.
- **Suspect file:** [backend/server.js](../backend/server.js#L30419)

### Role-gating that is **correct** (false alarms in raw report)
- orgAdmin → 403 on `/api/admin/{organizations,users,activity/recent,question-reports}` is correct ("Super admin only"). The smoke test's `expected` was too permissive.

## Quiz quality findings

### Top-level by source

| Source | Questions | Flagged (raw) | Real flagged | Caveat |
|---|---:|---:|---:|---|
| `frontend/data/quiz_data.js` | 396 | 63 (15.91%) | 63 | None |
| `backend/data/premade-questions.js` | 74 | 0 | 0 | Clean |
| `quizzes.txt` | 6 | 6 (100%) | **0** | Schema mismatch — not real |
| `frontend/Expanded/expanded.quizzes.bundle.js` | 3,588 | 521 (14.52%) | 521 | None |
| `frontend/New Exams/new_math_exams2.js` | 168 | 168 (100%) | **0** | Schema mismatch — not real |

**Schema-mismatch caveat:** `quizzes.txt` and `new_math_exams2.js` use a legacy schema where `answerOptions` is an array of strings and the correct answer is a sibling string field (`correctAnswer`). The quality classifier expects the canonical `{text, isCorrect, rationale}` object schema. These 174 "flagged" questions are not defective — they're a different format. Either:
- (a) Migrate both files to the canonical schema, OR
- (b) Add a normalizer to `classifyCoreQuestionQualityIssues` for the legacy shape.

### Real defect categories (across canonical-schema sources)

| Issue | Count | What it means |
|---|---:|---|
| `duplicate-rationales` | 376 | Multiple options share the same rationale text (the canned "This option does not accurately reflect..." copy). Mostly Social Studies expanded bundle. |
| `weak-rationale` | 274 | Rationale matches a known weak-pattern: empty, "this is incorrect", "consider the key details", etc. |
| `science-thin-rationale` | 97 | Science correct-answer rationale is too short or lacks an explanatory connector (because/when/which/etc.). |
| `short-correct-rationale` | 64 | Correct option's rationale < 24 chars. |
| `missing-stimulus` | 22 | Question references a passage/chart/graph but no passage/image is attached. |
| `duplicate-options` | 12 | Two answer options are textually identical (real defect — student can't pick the right one unambiguously). |

### Subject-tagging bug worth fixing

The expanded bundle tags **science visual-practice quizzes as `Social Studies`**:
- `science_img_physics_diagrams` → Social Studies (should be Science)
- `science_img_chemistry_diagrams` → Social Studies (should be Science)
- `science_img_science_graphs` → Social Studies (should be Science)

This will mis-route questions in subject filters, mastery tracking, and student-stat aggregation. Suspect file: [frontend/Expanded/expanded.quizzes.bundle.js](../frontend/Expanded/expanded.quizzes.bundle.js).

## Recommended fix order

1. **B1** (career-interests auth) — broken for everyone, single-line middleware swap.
2. **B2 + B3** (admin students search/export 500s) — admin tab is unusable.
3. Subject-mistagged science visual quizzes (one bundle edit, fixes mastery/stats fan-out).
4. Decide intent on **B5** (instructor reading admin reports). If intentional, rename routes off `/api/admin/`. If not, add admin gate.
5. **B4** (org-summary 400 for superAdmin) — depends on whether super-admin UI calls it.
6. Schema normalization for `quizzes.txt` + `new_math_exams2.js` (or migrate them).
7. Bulk-fix the canned-rationale Social Studies expanded quizzes (Cold War, Industrial America, New Nation, vocabulary 05–10, history-maps). These are content defects, not code.
8. Sweep `science-thin-rationale` (97) and `short-correct-rationale` (64) in `frontend/data/quiz_data.js` Science set.
9. Fix the 12 `duplicate-options` instances (data integrity — true ambiguity for students).
10. Address the 22 `missing-stimulus` questions (either attach the referenced passage/chart or rewrite the stem).
